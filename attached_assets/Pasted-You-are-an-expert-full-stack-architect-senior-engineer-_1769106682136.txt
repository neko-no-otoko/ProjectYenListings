PRODUCT GOAL:
A Zillow-like experience for “almost free” habitable akiya homes WITH LAND, in Japan, in English. Users search by criteria; we return a list + map, sortable and filterable, with normalized fields and a computed “Condition score” (1–5) based on listing description signals.

HARD FILTERS (must be enforced):
- Only abandoned/akiya-style homes (as best as can be inferred from sources/tags/categories).
- Must be “house + land” (not condo-only). If uncertain, label “Land: Unknown” and exclude by default unless user toggles “include unknown.”
- Purchase/transfer price MUST be ≤ 150,000 JPY. Listings explicitly “0 yen” / “0円” transfer are allowed.
- Assume “habitable even if rough” (leaks, cleaning, small utility issues). Exclude only if listing clearly indicates non-habitable / condemned / land-only / no structure / unsafe ruin, or if it violates the “home with land” requirement.

- Photo carousel (hotlink or permitted thumbnails), with attribution
- Condition score (1–5) + explanation bullets derived from text signals

3) Saved Searches + Alerts (MVP optional, but design for it)

NORMALIZATION / INGESTION PIPELINE:
For each enabled source, implement a “Source Adapter” module:
- discover(): returns candidate listing URLs (via sitemap/RSS/category pages if allowed)
- fetch(url): downloads HTML with polite headers, caching, backoff
- extract(raw): parses to structured fields (title, description, price, location, size, photos, tags). Use Cheerio for HTML parsing.
- normalize(extracted): maps fields to normalized schema and computes:
  - island from prefecture mapping table
  - has_land true/false/unknown from keywords (土地, 敷地, land area present, “宅地” etc.)
  - price_jpy parsed from “円 / yen / 0円 / 0 yen”
  - ldk parsed from strings (e.g., 2LDK)
  - condition_score from rules + optional LLM explanation step
- translate(): generate English title/summary/address if original is Japanese; cache by hash
- geocode(): if no coordinates, geocode; cache; store lat/lon
- nearest_airport(): compute haversine distance to nearest airport where is_major=true
- upsert listing and mark last_seen_at

DEDUP STRATEGY:
- Use a fingerprint hash of (normalized address + house_sqm + land_sqm + price_jpy + lat/lon rounded) + title similarity.
- Use trigram similarity on title + address and distance threshold (e.g., within 250m).
- Keep canonical listing but retain per-source variants.

CONDITION SCORE (1–5) REQUIREMENTS:
Implement a deterministic baseline scorer (no LLM needed):
- 5 = move-in ready / renovated / “good condition”
- 4 = livable, minor repairs/cleaning
- 3 = needs work (leaks mentioned, utilities issues) but still framed as usable
- 2 = major repairs, but still a standing home
- 1 = ruin/unsafe/condemned (exclude by default as “not habitable”)
Then optionally add an LLM enrichment step that writes:
- description_en summary (3–6 sentences)
- condition_reasons_en bullet list
The LLM prompt must NEVER invent facts; it must only rephrase and translate. It must output JSON.
