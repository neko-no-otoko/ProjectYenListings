## Technical Integration Notes

### URL Patterns:

**At Home Subdomain Pattern:**
```
https://{city-name}-c{jis-code}.akiya-athome.jp/     # City
https://{town-name}-t{jis-code}.akiya-athome.jp/     # Town
https://{village-name}-v{jis-code}.akiya-athome.jp/  # Village
```

**Examples:**
- 函館市: `hakodate-c01202.akiya-athome.jp` (c = city)
- 木古内町: `kikonai-t01334.akiya-athome.jp` (t = town)
- 中札内村: `nakasatsunai-v01638.akiya-athome.jp` (v = village)

### Data Fields (Standardized):
1. Property ID
2. Property Type (売戸建, 売土地, 貸家)
3. Price (万円)
4. Address
5. Layout (3DK, 2LDK, etc.)
6. Land Area (m²)
7. Building Area (m²)
8. Construction Date
9. Images
10. Contact Information

### Scraping Considerations:
- At Home: Standardized structure, but subdomain-per-municipality
- Regional portals: Custom HTML, requires per-site parsers
- Municipal sites: Highly variable, maintain separately
